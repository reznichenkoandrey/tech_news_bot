## TL;DR
Anthropic випустила Claude Opus 4.5 — найпотужнішу модель для coding, агентів і computer use, яка перевершує всі конкуруючі моделі на реальних інженерних завданнях, при цьому використовуючи на 48-76% менше токенів і коштуючи $5/$25 за мільйон токенів.

## Ключові тези

- **Новий стандарт для coding**: На тесті SWE-bench Verified досягла 56.7% (найкращий результат у світі), на внутрішньому тесті Anthropic для інженерів набрала вищий бал, ніж будь-який людина-кандидат за всю історію.
- **Різке зниження вартості**: Ціна знизилася до $5/$25 за мільйон токенів при вищій якості, модель використовує на 48-76% менше output токенів для аналогічних результатів порівняно з Sonnet 4.5.
- **Новий параметр effort**: Дозволяє розробникам контролювати баланс між швидкістю/вартістю та якістю — на середньому effort досягає результатів Sonnet 4.5, використовуючи на 76% менше токенів.
- **Найбезпечніша модель**: Показує найкращу стійкість до prompt injection атак серед усіх frontier моделей, продовжує тренд Anthropic на більш aligned системи.
- **Продуктові оновлення**: Claude Code тепер у desktop app з Plan Mode, Claude для Chrome доступний усім Max користувачам, Claude для Excel у бета для Max/Team/Enterprise, необмежені conversation у додатках.

## Deep dive

Claude Opus 4.5 представляє суттєвий зсув у можливостях AI для практичних завдань. Anthropic позиціонує модель не просто як покращення метрик, а як якісний стрибок у "розумінні" складних, багатокрокових завдань.

**Чому це важливо для розробників**

Модель демонструє рідкісну комбінацію: вища якість при нижчих витратах. На benchmark SWE-bench Verified (реальні GitHub issues) вона досягла 56.7% — абсолютний рекорд. Внутрішній тест Anthropic особливо показовий: Opus 4.5 з parallel test-time compute перевершила всіх людей-кандидатів на складному 2-годинному іспиті з performance engineering. Це не теоретична метрика — клієнти як GitHub, Cursor, Warp повідомляють про 15-20% покращення на внутрішніх benchmark і зниження помилок на 50-75%.

Новий параметр effort змінює економіку використання LLM. На середньому рівні модель досягає попередніх top результатів, витрачаючи на 76% менше токенів. На максимальному effort перевершує їх на 4.3 п.п., але все одно використовує на 48% менше токенів. Це означає, що Opus-рівень можливостей тепер доступний для щоденних завдань, а не тільки критичних кейсів.

**Аспект безпеки**

Anthropic приділяє особливу увагу alignment. Opus 4.5 показує найкращу стійкість до prompt injection серед frontier моделей — критично важливо для enterprise застосувань, де модель працює з чутливими даними. Це результат цілеспрямованої роботи над Constitutional AI та robustness training.

**Цікавий нюанс з benchmark**

На τ2-bench модель "провалила" тест, бо знайшла творче рішення для клієнта авіакомпанії: спочатку upgrade cabin (дозволено для basic economy), потім зміна рейсу (дозволено для non-basic). Benchmark очікував відмову, але Claude показала саме той тип problem-solving, який роблять хороші люди-агенти. Anthropic визнає цю напругу між "проходженням тесту" та корисною поведінкою.

**Практичні висновки**

Для команд це означає можливість автоматизувати складніші workflow — від code refactoring через кілька кодових баз до 30-хвилинних autonomous coding сесій. Комбінація з context management, memory capabilities та multi-agent orchestration (де Opus 4.5 ефективно керує subagents) дає +15 п.п. на deep research завданнях. Зниження ціни робить Opus основною моделлю замість "премієм варіанту для особливих випадків".
